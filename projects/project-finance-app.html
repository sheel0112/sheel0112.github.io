Stacked – Personal Finance & Budgeting App

Product Manager • Fintech • ML-driven spend insights

Summary

  • Created a lightweight personal finance tool for young adults and couples.
  • Auto-categorises spend and surfaces nudges + monthly insights.
  • Designed scalable data contracts for bank APIs and analytics pipelines.

Context & Problem

Many budgeting tools overwhelm users with manual effort. The goal was to create a simple, low-maintenance system that automatically categorises spend, shows trends, and provides actionable suggestions.

Constraints

  • Needed to integrate cleanly with existing bank APIs (TrueLayer).
  • Designed for low manual input — automation-first.
  • Simplified UI to avoid overwhelming users.

Approach

Understanding the user

Interviews showed that users want “just enough insight” — not charts, but clarity. This shaped the feature set around simple summaries and nudges.

Systems thinking & architecture

I defined end-to-end data flow from API ingestion → categorisation → ML tagging → insights engine → UI layer. The focus was reliability and fast iteration.

Delivery

Built UI flows, data schema, and release plan. Added categorisation testing and a fallback manual tagging system to handle edge cases.

Outcome

  • Clear, actionable monthly summaries.
  • Automatic insights based on recurring vs one-off patterns.
  • Simple interface that works on mobile and desktop.

Next Steps

Expand the nudges engine, refine ML clustering, and introduce envelope budgeting for couples trying to split expenses.
