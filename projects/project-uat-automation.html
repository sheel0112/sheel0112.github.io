MLB UAT Automation Platform

Automation · Pricing Validation · Internal Tooling · Risk Controls

Executive summary

I designed and delivered a full UAT automation suite for MLB pricing, replacing manual testing with a scalable platform that performs complete market validation in minutes. This tool reduced regression errors by 80% and cut testing time by 50%, becoming the blueprint for automation across NBA, NFL, and future risk systems.

Strategic context

MLB pricing creates thousands of market-level combinations. Historically, each release involved:

  • large Excel sheets and manual comparison
  • unrepeatable test flows across operators
  • inconsistent testing depth between testers
  • no regression framework

Operational risk increased with every model update — the business needed automation, visibility, and confidence.

The problem

The core challenge was scale. Manual testing couldn’t keep up with:

  • operator-specific behaviour differences
  • complex ladder structures
  • multi-market dependencies
  • settlement and pricing edge cases

Pricing updates became slow, risky, and resource-heavy.

Constraints

  • No guaranteed engineering resources
  • Needed to run entirely on static Athena datasets
  • Validation rules had to mirror real model behaviour
  • Tool had to work across operators, even with structural differences

My role

This was a full-stack product ownership initiative:

  • mapped complete MLB pricing lifecycle and rules
  • designed the validation framework and checks
  • built the initial Python automation engine
  • managed transition to a React + SignalR web app
  • defined KPIs, coverage, and adoption goals
  • collaborated with traders, engineering, and analysts daily

What I delivered

  • Rules engine validating alignment, ladders, envelopes, correlation, and settlement
  • Multi-operator validation modes for BetMGM, Entain, etc.
  • Excel + CSV reporting with automatic exception highlighting
  • Regression test library to standardise model release QA
  • Web app interface enabling multi-user access and visibility
  • Framework for future automation across all sports

Impact

  • 50% faster test cycles
  • 80% reduction in missed regression errors
  • Full-market coverage (instead of spot-checking)
  • Safer, more confident mid-season model updates
  • Platform adopted across multiple sports

Relevance to fintech & banking

This project mirrors how financial institutions build risk controls:

  • pricing envelopes ≈ risk limits
  • settlement behaviour ≈ trade lifecycle modelling
  • multi-operator differences ≈ multi-region regulatory variance
  • validation checks ≈ pre-trade & post-trade controls

It demonstrates comfort with automation, control design, and data-driven risk processes.
